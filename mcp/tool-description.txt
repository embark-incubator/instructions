Semantic code search over the indexed codebase.
Takes a natural-language query, returns a ranked list of code snippets (file path, line numbers, code) relevant to the query.
Ranking considers semantic meaning AND symbol / file / directory names — the query can describe what you want even if you do not know the exact names.

Use this tool to answer questions whose shape is:
- "How does X work?"
- "Where in the codebase does behaviour Y live?"
- "Which components are involved in Z?"
- "What pattern is used for W?"
- "What is the role of X in Y?"
- "Where is the test suit for X?"

Use Grep / Glob / Read instead when the question is:
- "Where is this exact symbol / string / import defined?"
- "Which files match this name or extension?"
- "Open this specific path."

The deciding factor is the shape of the question, not whether you already know a symbol name.
For example:
- "Find EmailService" is a location question (Grep).
- "How does EmailService coordinate with the retry controller during partial failures?" is a behaviour question even though EmailService is named — use
this tool (code search).

If a query returns nothing useful, reformulate the query (different vocabulary, drop the path filter, split compound questions into separate calls).
Do not give up after one miss.
